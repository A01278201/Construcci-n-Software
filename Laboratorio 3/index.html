Preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    Internet es la red física que conecta computadoras en todo el mundo, mientras que la Web es un sistema de información que utiliza Internet para acceder a documentos y páginas mediante navegadores.

  2. ¿Cuáles son las partes de un URL?
    Protocolo (http/https), nombre de dominio, puerto (opcional), ruta del archivo, parámetros y fragmentos (hash).

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    - GET: Obtener datos del servidor.
    - HEAD: Obtener solo los encabezados de la respuesta.
    - POST: Enviar datos al servidor para crear algo nuevo.
    - PUT: Reemplazar completamente un recurso.
    - PATCH: Actualizar parcialmente un recurso.
    - DELETE: Eliminar un recurso.

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    Se debe usar POST, porque los datos se envían en el cuerpo de la solicitud y no quedan visibles en la URL, lo cual brinda más seguridad.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    Se utiliza el método GET.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    Significa que la solicitud fue exitosa. No ocurrió ningún error.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    Sí, porque un error 404 indica que una página solicitada no existe. El desarrollador debe revisar enlaces rotos o rutas mal configuradas.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    Sí, ya que un error 500 indica un fallo en el servidor. Esto generalmente está relacionado con errores en el código del lado del servidor.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    Significa que ya no se recomienda su uso porque hay alternativas mejores. Ejemplos: <font>, <center>, <bgsound>.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    HTML5 incorpora etiquetas semánticas (<header>, <footer>, <article>), soporte para multimedia (<audio>, <video>), formularios mejorados y es más amigable con dispositivos móviles.

  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    Estructura: <table>, <thead>, <tbody>, <tfoot>, <tr>, <th>, <td>.
    Estilo: Se aplica con CSS (bordes, colores, alineación, espaciado, etc.).

  12. ¿Cuáles son los principales controles de una forma HTML5?
    <input> (text, email, number, date, etc.), <textarea>, <select>, <button>, <label>.

  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?
    La mayoría de los navegadores modernos como Chrome, Firefox, Edge o Safari tienen soporte casi completo para HTML5.

Preguntas Laboratorio 3

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    Mi recomendación es usar !important solo cuando sea absolutamente necesario, por ejemplo, para sobrescribir estilos de librerías externas. Usarlo en exceso complica el mantenimiento del código, ya que rompe la cascada natural de CSS.
  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    Porque una mala elección puede afectar la legibilidad del contenido o ralentizar la carga del sitio si el archivo es muy pesado. También se debe verificar que se vea bien en diferentes tamaños de pantalla.
  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?
    Usar unidades relativas como % o em cuando se quiera responsividad y escalabilidad, por ejemplo en diseño adaptable. Reservar px para elementos que requieren dimensiones fijas, y evitar pt en pantallas web, ya que está pensado para impresión.
  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    Porque reduce el tamaño del archivo al eliminar espacios, saltos de línea y comentarios, lo que hace que se cargue más rápido en el navegador y se mejore el tiempo de respuesta del sitio.
